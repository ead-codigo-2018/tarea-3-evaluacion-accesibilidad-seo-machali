I. Municipalidad de Machalí: Informe de accesibilidad y optimización

Páginas evaluadas: inicio, DIDECO/Egis, Trámites/preguntas frecuentes, Trámites/trámite fácil

1) Evaluación según pautas WCAG 2.0

Perceptible

  • 1.1: Proporcionar alternativas textuales para todo contenido no textual de modo que se pueda convertir a otros formatos que las personas necesiten.

    Inicio: no hay etiquetado alt en los íconos.

    Inicio: hay botones que son imágenes y su etiquetado alt no es pertinente, además no es responsive.

    Inicio: no hay etiquetado alt en los botones, algunos tienen link vacío.

    Egis, preguntas frecuentes, trámite fácil: no hay etiquetado alt en las imágenes (los cuales son botones), direccionan a un link vacío.

    Egis, preguntas frecuentes, trámite fácil: el etiquetado alt está en inglés.

  • 1.2: Proporcionar alternativas para los medios tempodependientes.

    No aplica.

  • 1.3: Crear contenido que pueda presentarse de diferentes formas sin perder información o estructura.

    • Inicio, egis, preguntas frecuentes, trámite fácil: La información del sitio no está disponible para que todo tipo de público la lea. No hay soporte auditivo ni otras opciones para discapacitados.

  • 1.4: Facilitar a los usuarios ver y oír el contenido, incluyendo la separación entre el primer plano y el fondo.

    Inicio, egis, preguntas frecuentes, trámite fácil: el tamaño tipográfico es menor al recomendado para sitios web (14 en vez de 16), la fuente es muy liviana y de un gris que no hace mucho contraste con el fondo blanco (en el cuerpo y algunos botones).

    Inicio, egis, preguntas frecuentes, trámite fácil: el footer no es responsive, así que el texto se descuadra y se sale de la caja en donde debería estar, perdiendo contraste.

    Egis (y todo el botón extendible DIDECO): el rojo utilizado en el botón DIDECO es demasiado saturado y vibrante.

Operable

  • 2.1: Proporcionar acceso a toda la funcionalidad mediante el teclado.

    Inicio, egis, preguntas frecuentes, trámite fácil: el sitio es navegable por tab, pero no indica la posición del "cursor".

  • 2.2: Proporcionar a los usuarios el tiempo suficiente para leer y usar el contenido.

    Inicio: el banner automático se mueve demasiado rápido (3 segundos), por lo que no se alcanza a leer cada etiquetado antes de que se mueva.

  • 2.3: No diseñar contenido de un modo que se sepa podría provocar ataques, espasmos o convulsiones.

    Inicio, egis, preguntas frecuentes, trámite fácil: no hay riesgos en ningún lugar.

  • 2.4: Proporcionar medios para ayudar a los usuarios a navegar, encontrar contenido y determinar dónde se encuentran.

    Preguntas frecuentes, trámite fácil: Hay breadcrumbs, pero no están completos (ejemplo egis está bien, ejemplo preguntas frecuentes está mal).

Comprensible

  • 3.1: Hacer que los contenidos textuales resulten legibles y comprensibles.

    Inicio, egis, preguntas frecuentes, trámite fácil: el puntaje, peso y color tipográfico del cuerpo de texto y algunos botones imposibilita la legibilidad para personas mayores o con dificultades visuales.

  • 3.2: Hacer que las páginas web aparezcan y operen de manera predecible.

    Inicio, egis, preguntas frecuentes, trámite fácil: existen botones con contenido vacío y botones repetidos.

  • 3.3: Ayudar a los usuarios a evitar y corregir los errores.

    Trámite fácil: al ingresar datos como el RUT, el sistema indica cómo escribirlo, indica si se cometió un error, e indica cómo editarlo de manera correcta (en la barra para el RUT no permite poner letras excepto la k). En la sección del rol, esto no pasa.

Robusto

  • 4.1: Maximizar la compatibilidad con las aplicaciones de usuario actuales y futuras, incluyendo las ayudas técnicas.

    Inicio, egis, preguntas frecuentes, trámite fácil: el html y css no son válidos, entre ambos hay 61 errores.

1.2) Mejoras de accesibilidad propuestas

  • Etiquetado alt correcto y en español.
  • Quitar la publicidad del inicio (la cual no es responsive)
  • Evitar los botones imagen.
  • Mejora en la tipografía (puntaje, color, peso).
  • Mejora en los breadcrumbs (orden correcto).
  • Proporcionar medios para que el sitio sea accesible para todo tipo de público.
  • Evitar colores demasiado saturados y vibrantes que dañan la vista.
  • Ralentizar el movimiento de los banners automáticos.
  • Evitar la repetición innecesaria de contenido, evitar links vacíos.
  • Mantener el código válido.

2) Evaluación de SEO

  • No hay ningún título en h1.
  • La metadescripción es muy corta (48 caracteres).
  • La página de inicio no tiene metadescripción ni meta-keywords.
  • La url utiliza guiones bajos en vez de guiones altos.
  • El sitio no está optimizado para móbiles.
  • El sitio no cuenta con una página personalizada de error.

2.1) Mejoras de SEO propuestas

  • Mejorar y agregar las metadescripciones correspondientes
  • Cambiar los guiones bajos por guiones altos en la url.
  • Optimizar el sitio para móbiles.
  • Añadir una página personalizada de error.

2.2) Análisis de metadatos y datos estructurados

  • Ninguna página tiene metadato de idioma.
  • Los metadatos estan desordenados.
  • El sitio no tiene errores de datos estructurados.
